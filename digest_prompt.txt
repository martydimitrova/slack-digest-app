# Digest generation prompt
# Edit freely. Variables substituted at runtime:
#   {window_start}   {window_end}   {run_id}   {output_path}
#   {sections_block} {dm_line}
# Note: {{ and }} are literal { } in the JSON schema below.

Generate a Slack digest for the period {window_start} to {window_end}.

BREVITY: whole digest readable in under 10 min (~1500 words max). Scale DOWN for low activity. Never pad.
OMIT: reaction counts, emoji tallies, "X people reacted", ack counts, and any sentence whose only content is how many people saw something.
EMPTY CHANNELS: for channels with no messages in the window, include them with key_points: ["No activity."] and link: null — never omit silently.
KEY_POINTS: at most 3 bullets per thread regardless of depth. One sentence each.
LINKS: always populate the link field for every thread that has a root message. Use the Slack archive URL format https://slack.com/archives/CHANNEL_ID/pTIMESTAMP. For low depth, include the top 1–3 most important threads per channel as reference links even though the section uses a prose summary.

DEPTH:
- high   → all channels; all notable threads summarised individually; participants listed per thread
- medium → all channels; a few aggregated conversations each; surface anything with a date/deadline
- low    → section_summary: one aggregated paragraph for the whole section; channels still list top threads with links for reference (key_points may be a single brief label or empty)

SECTIONS:
{sections_block}

STEPS:
1. For each section, treat its Customisation (if set) as instructions that extend or override the defaults for that section — e.g. "translate to English", "focus on incidents", "skip bot messages".
2. For each channel, read messages in the time window using available Slack tools.
   If a tool call fails, wait briefly and retry up to 2 times. If it still fails, set
   key_points: ["Could not read — tool error after 2 retries."] and link: null for that channel.
3. Summarise per section according to depth. Print progress as you go:
   "Reading #channel-name..."  /  "Summarising Section Name..."  /  "Writing digest..."
4. Write the final result as valid JSON to: {output_path}
5. JSON shape (use exact run_id/window values given):
{{
  "run_id": "{run_id}",
  "generated_at": "<ISO UTC>",
  "window_start": "{window_start}",
  "window_end": "{window_end}",
  "sections": [
    {{
      "name": "<section name>",
      "depth": "<high|medium|low>",
      "section_summary": "<low depth only: one aggregated paragraph. empty string for high/medium>",
      "channels": [
        {{
          "name": "<channel name without #>",
          "threads": [
            {{
              "link": "<https://slack.com/archives/CHANNEL_ID/pTIMESTAMP or null>",
              "participants": ["@user1", "@user2"],
              "key_points": ["..."]
            }}
          ]
        }}
      ]
    }}
  ]
}}
{dm_line}
